Use cases for self-checking quizzes

Fill in the expression

Figure 0.
Figure 1.

System under description: the web client.

Actors: The student, the RTM server

Precondition: The quiz question is in its initial state. An RTM is connected.

Main Flow

  1. The student reads the question text and fills the expression text box. (See Figure 0).
  2. Once the value of the expression text box is valid, the system enables the start button.
  3. The student clicks on the start button.
  4. The client reveals an execution panel with some initial values for the input variables already filled in.
  5. The client constructs a new source text based on the template source, the value of the expression text box, and the initial values for the input variables. It then sends a loadString message to the RTM server.
  6. The RTM server sends a reply.
  7. Assuming the file compiled without error (state COMPILED), the client and uses the RTM executes the source code up to the beginning of the expression. The expression display should show the expression in its initial state. (see Figure 1)
  8. At this point, the state is READY or EXECUTION_COMPLETE, or EXECUTION_FAILED. The goForward button should be enabled if the state is READY. The goBack button should be enabled. The start button has been relabelled restart and is enabled.
  9. The user has several options at this point
    1. Click on the goForward or goBack button.
      1. The system sends a go or goBack request
      2. The RTM sends a reply.
      3. The client updtates the expression display.
      4. Back to to step 8.
    2. Change the expression text box or an initial value box.
      1. The client disables the goForward and goBack buttons
      2. The user can make further changes to the expression text box or the initial value boxes. Throughout this editing, the start button is enabled iff the expression box and initial value boxes pass the validity checks.
      3. The user clicks on the start button.
      4. Back to step 5.
    3. Click on the start button.
      1. Back to step 5
    4. Close the execution panel.
      1. Back to step 2.

Postcondition: None. The above is an infinite process. It ends when the student moves to another question or leaves the page the question is on.

Alternative paths:

  • At step 6: if there is a compile time error, the execution panel is closed, the error message should be displayed to the user, and we go back to step 2. It might be helpful to display the line of code that caused the error.
  • At step 8: if the state is EXECUTION_FAILED, an error message should be displayed to the user.

Notes:

  • The expression is valid if it is not empty.
  • Initial values are valid if they can be interpreted as C++ constants of the appropriate type.
