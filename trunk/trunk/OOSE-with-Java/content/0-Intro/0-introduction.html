Java

  • Java is an imperative, object-oriented, programming language.
  • It is fairly simple and comes with a great set of libraries.
  • It is very portable across operating systems and hardware platforms
    • It is compiled to a machine independent virtual machine code.
    • The virtual machine code is either interpreted or locally compiled to real machine code.
    • Libraries are portable (SE for large machines, ME for phones etc)
  • Java is unusually safe.
    • Many categories of errors are detected by the compiler.
      • Example: Failing to initialize a variable.
    • Many categories of errors are detected at run-time rather than leading to undefined behaviour.
      • Example: Misuse of null pointers.
    • Some categories of errors are simply impossible to make in Java.
      • Example: Using a pointer to a deallocated object.
  • Because of its safety and portability, Java code is suitable for inclusion in web content (applets).
  • Java supports concurrency
    • We will not use this feature in this course.
  • There are a range of quite decent programming tools for Java.
    • We will use Eclipse

Object-Oriented Design

Object-oriented design involves thinking of systems as collections of "objects"

  • Each object isolates part of the program state from the other objects.
  • A design for an object is called a "class"
  • The design of the program is thus in terms of "classes"
  • Each class represents the design for a set of objects with identical behaviour and state sets.

Using the JDK Compiler

Let's look at a second program and see how we can compile and run it using Sun's Java Development Kit (JDK).

My examples are based on the Windows OS, but the same applies to Unix with the details changed a bit.

I'll assume that the JDK is installed and that the JDK's bin directory is on the PATH environment variable. For this course you will need version 5.0 at least (this is also known as version 1.5).

C:\>

We use a text editor such as JEdit, UltraEdit, WinEDT, gvim, or even notepad, to put this program in a file called Greetings.java.

The file name matches the name of the class.

C:\>
C:\>type Greetings.java
/******* A Friendly Program ********/                
final class Greetings {
   public static void main(String [] args){
      System.out.println( "Greetings earthlings" ) ;
   }
}
C:\>

The javac command compiles the program, producing a file called Greetings.class.
