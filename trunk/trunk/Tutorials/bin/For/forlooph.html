For-loop tutorial

This tutorial explains the working of a for-loop with examples.

Example 1: Checking if a number is prime or not.It is assumed that the number is greater than 1.The function is called isPrime() which returns boolean variables true or false.

First we declare a bool variable b and assign it to the return value of the isPrime() function. isPrime() returns true if the number passed to it is prime and returns false otherwise. We pass number 7 to the isPrime() function to verify that it is prime .

In the isPrime() definition: The number is copied into a local variable called num.

A bool variable called isPrime is declared and initialised to true.

The for loop: In the first iteration a variable div local to the loop is created and intialized to 2.Now we divide num by div while incrementing div (div++) in each iteration. The remainder of num/div (given by num%div) is logically ANDED with isPrime. So if the result of any division is 0 i.e. num is divisible by a div then isPrime becomes false because true&&0 is false else it remains true.

div=2::num=7::num%div=1::isPrime=true&&2=true

div=3::num=7::num%div=1::isPrime=true&&1=true

Note the values of a and b. They remain unchanged. Now we pass a and b to the Pass-by-reference version of the intSwap() function.Notice the function call. It is same as the call to the Pass-by-value version.

Carefully look at the function definition. In the first line two reference variables arg1 and arg2 are created pointing to actual variables a and b respectively. References are just separate names for the variables they point to.

Memory is allocated for the temp variable.

Notice that 'temp=arg1' copies value 5 into temp, because arg1 is a reference to a or just another name for variable a so temp=arg1 assigns temp the value of a which is 5.

Similarly, arg1=arg2 copies value of b into a. because arg2 is reference to b and arg1 is reference to a.

Finally, the value of a stored in temp through arg1 is copied back into arg2 which is nothing but b.

The function returns and all the local variables including the references are destroyed. Now note the values of a and b. They are swapped!
