Object

java.lang.Object is a class that is at the root of the "extends" tree. That is, every class extends from the Object class either directly or indirectly.

Even array types extend the Object class.

There are some things you can do with all objects

  • boolean equals( Object y )
    • x.equals(y) is true if x and y represent the same thing
  • String toString()
    • Return a string representation for the object
  • ...

String

java.lang.String is a class of immutable strings (sequences of characters).

String literals are really objects of class String. E.g.

"abc".equals( x )

looks unusual, but it sends the equals(x) message to an object of type string.

Strings can be concatenated (producing new strings). For example

"(".concat( x.toString() ).concat( ")" )

gives the string representation of object x inside parentheses.

The + operator is overloaded to mean concat so

"to" + "get" + "her"

produces the object "together".

Furthermore if one operand of + is a string and the other isn't, the other will be converted to a string (using toString for objects). So you can write

System.out.println( "The value of i is " + i ) ;

which (if i is an int) is the same as

System.out.println( "The value of i is ".concat( Integer.toString(i) ) ) ;

System

java.lang.System is used for its static fields and method. No need to instantiate it.

  • System.in, System.out, System.err
    • These are three static fields that give you access to the standard input, standard output, and standard error streams.
    • When you start a java program from the command prompt (cmd.exe in windows or sh/bash/csh etc in Unix) there are, by default, connected to the keyboard and the terminal.
    • System.in is of type java.io.InputStream
    • System.out and System.err are of type java.io.PrintStream
  • System.exit( int status )
    • Stops the program immediately. Use 0 for success and 1 for failure.

Byte, Short, Integer, Long, Float, Double, Character, Boolean

These classes provide static methods and constants relevant to the corresponding primitive types.

For example

  • Integer.MAX_VALUE and Integer.MIN_VALUE
    • give maximum and minimum values for the int type.
  • String Integer.toString( int i )
    • as we've seen formats an int as a String.
  • double Double.parseDouble( String s )
    • converts a String to a double
    • Throws NumberFormatException if the format is wrong.

Also you can use these classes to create immutable Objects that can be used where primitive values can not.

For example the ArrayList class holds references to Objects. So we can't store int values on an ArrayList. But we can do the following

ArrayList<Integer> intList = new ArrayList<Integer>() ;
  .
  .
  .
  for( int i = 0 ; i < M ; ++i )
      intList.add( new Integer(i) ) ;
  .
  .
  .
  int k = intList.get(0).intValue() ;

In many cases, the Java compiler automatically inserts such allocation expression and calls to intValue() and similar methods. This is called autoboxing -- read about it.

Input and Output

File

File names are represented by objects of type java.io.File.

For example

new File( "myData.txt" ) ;

creates a File object representing a file named myData.txt in the current working directory.

For example

(new File(".")).getCanonicalPath()

gives the full name of the current working directory.

Reader and Writer and binary I/O

java.io.Reader and java.io.Writer provide simple interfaces for human readable files (text files).

java.io.DataInput and java.io.DataOutput provide nice interfaces for reading and writing binary files in a portable format.

Formatted I/O

To print human readable files, use java.io.PrintWriter

For example

File aFile = ... ;
PrintWriter pwriter = new PrintWriter( aFile ) ;
pWriter.print( anInt ) ;
pWriter.print( aString ) ;
pWriter.printf("Unable to open file '%1$s': %2$s",
               fileName,
               exception.getMessage());
pWriter.printf("Today is %1$tB %1$te,%1$tY",
               new GregorianCalendar() ) ;

To read formatted data use java.util.Scanner

File aFile = ... ;
Scanner scanner = new Scanner( aFile ) ;
int anInt = scanner.nextInt() ;
double aDouble = scanner.nextDouble() ;

Scanners generally skip over spaces and newlines.

Graphical User Interfaces

Java provides two libraries for GUIs.

  • The AWT is the Abstract Windowing Toolkit.
    • It is "abstract" as it abstracts away from the details of the underlying windowing system (Windows, X, Mac)
  • "Swing" is more modern and even more portable.

A simple swing program

Swing GUIs are made up of objects that are "Components"

  • JLabel: contains fixed text
  • JTextField: editable text
  • JTextArea: multi-line editable text
  • JEditPane: multi-line formatted editable text
  • JButton: clickable areas
  • JMenuItem: like a button, but intended for menus
  • JTree: trees (like in Windows Explorer)
  • JTable: tables

Some components are "containers" that other components can be added to

  • JFrame: primary window of applications
  • JDialog: windows that are tied to others
  • JMenuBar: menu bars
  • JMenu: menus
