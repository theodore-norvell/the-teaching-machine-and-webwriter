Statements

Commands in Java are called statements. We've already seen one form of statement, which is called an expression statement.

Statement --> Expression ;

Note Not every kind of expression can be used in an expression statement, but the kinds that make sense can. End of Note.

Expression Statements

Expression statements are executed simply by evaluating the expression, presumably because the expression has some side effect.

Skip Statements

A skip statement consists of only a semi-colon and does nothing at all

Statement --> ;

Another way to write a statement that does nothing at all is

 {}

I prefer {} to ;

If Statements

If statements allow a choice between two alternative statements based on a boolean value. A two-tailed if statement has the form

Statement --> if( Expression ) Statement else Statement

The expression must be boolean and if it evaluates to true, then the first statement is executed, otherwise the second statement is evaluated.

We can also write a one-tailed if-statement

Statement --> if( Expression ) Statement

In this case, if the expression evaluates to false, nothing is done, as in the skip-statement.

In effect a statement

if( E ) S

means

if( E ) S else ;

Note Because Java has both one-tailed and two-tailed if statements there is a question of associativity. A statement

if( E0 ) if( E1 ) S0 else S1

Could be understood in two ways. The rule is that each "else part" is attached to the most recent "if" possible. In the example S1 will be executed only if E0 evaluates to true and E1 to false.

End of Note.

Blocks

In some cases we want to execute a sequence of statements.

In such a case we can use a block

Statement --> Block
Block --> { ( BlockMember )* }
BlockMember--> Statement

[Here I'm using some new meta-notation to describe the syntax. When I put something between the symbols ( and )* that means that there can be zero or more repetitions of that something.

So the above definition says that

  • a Statement can consist of a Block,
  • a Block can consist of a "{" followed by zero or more BlockMembers, followed by a "}", and that
  • a BlockMember can be a Statement.]

Here are some examples of statements that are blocks

  • { }
  • { c = 3 ; }
  • { c = 3 ; d = 4 ; }

Usually, a block is executed by executing each of the statements in it, starting with the first and ending with the last.

  • Thus it is the sequential composition of the statements in it.
  • However, that is assuming that each of the statements completes normally. It is also possible for statements to complete abruptly. We'll address abrupt completion later in this course.

The following is a reasonable if-statement. Each tail is a block 

if( a == b )
{
    c = 3 ;
    d = 4 ;
}
else
{
    c = 4 ;
    d = 5 ;
}

Aside on indentation

By the way, in Java, newline characters and blank characters are equivalent in most contexts and, in fact, optional in most cases. So a completely equivalent statement is this:

if(a==b){c=3;d
=4;}else{c=4;d
=5;}

However, it is good style to use blanks and newlines to highlight the structure of your programs by indenting nested elements more than outer elements.

Local variable declarations

Besides statements, another thing you can put in blocks is local variable declarations.

Simplifying slightly, the syntax is

BlockMember --> VariableDeclaration
VariableDeclaration --> Type Identifier = Expression ;

So for example

int balance = 0 ;

is a local variable declaration and can appear within a block.

The meaning of a local variable declaration is that it declares a new variable for use within a block.

The scope of the variable starts at its declaration and ends at the end of the enclosing block.

The meaning of a local variable declaration is that

  • the variable is created on the stack
  • the expression is evaluated
  • the value of the expression is assigned to the variable

The variable is then available for use or reassignment throughout the remainder of the block.

At the end of the block, the local variable ceases to exist.

Principle: Local variables cease to exist after the block in which they were declared is complete.

 

The assignment sign and the expression in the local variable declaration can be left out. To express this, I'll introduce some new meta-notation.

When I enclose something between ( and )? in grammar rule, it means that that something is optional. So we have:

VariableDeclaration --> Type Identifier ( = Expression )? ;

Here is an example

double distance ;

However, my style recommendation is to make the scope of each local variable no bigger than needed and usually this means that you can initialize the variable as part of its declaration.

Of course computers are only really useful because the can do the same thing over and over, with different data.

while statement

The syntax of the while statement (or "while loop") is

Statement --> while( Expression ) Statement

The execution of a while statement

while( B ) S

is as follows

  • (0) B is evaluated.
  • (1) If B evaluates to false, then nothing is done.
  • (2) If B evaluates to true, then S is executed and then we return to step (0)

for statements

for statements (or for loops) are essentially an abbreviation. The last two examples had the same general form of:

{ D while( B ) { S E; } }

where D declares a variable, B tests it, and E updates it.

for loops give us a slightly more compact way to write the same thing. Using a for loop, we would write

for( D B ; E ) { S }

The syntax is as follows

Statement --> for( VariableDeclaration Expression ; Expression ) Statement

When the variable is an integer, the update is often to increment it by one, e.g.

i = i + 1

There is a convenient abbreviation for this, which is

++i

Sometimes, however we decrement by one:

i = i - 1

This can be abbreviated by

--i

Using this abbreviation and a for loop, our triangular number computation becomes

The word counter example can be written 

int count = 0 ;
for( Scanner scanner = new Scanner( System.in ) ;
             scanner.hasNext() ;
             scanner.next() )
    count = count + 1 ;
