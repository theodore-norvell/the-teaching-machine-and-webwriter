Enumerations

Enumeration types are a special kind of class. Only a fixed set of objects are instances of the class and each of these has a name.

A declaration

enum Suit {
	HEART, CLUB, DIAMOND, SPADE
}

is, in essence, very similar to a declaration

class Suit {
    private Suit() {}
    static final public Suit HEART = new Suit() ;
    static final public Suit CLUB = new Suit() ;
    static final public Suit DIAMOND = new Suit() ;
    static final public Suit SPADE = new Suit() ;
}

In each case, there is a fixed set of 4 instances of the Suit type; no more instances of the type can be created.

This can be useful for creating types that represent small set of a distinct values.

Some more examples

enum Face {
	ACE, TWO, THREE, FOUR, FIVE,
	SIX, SEVEN, EIGHT, NINE, TEN,
	JACK, QUEEN, KING 
}
enum Gender {
	MALE, FEMALE
}
enum AgeCategory {
	INfANT, CHILD, ADULT, SENIOR
}
enum ElevatorState {
	STOPPED, ASCENDING, DESCENDING
}

Enumeration types have a few more useful characteristics.

For example

Suit.HEART.toString()

results in "HEART"

The .values() static method returns an array of all values.

Therefore, we can loop through all members of an enumeration

for( Suit s : Suit.values() ) 
    for( Face f : Face.values() ) 
        deck.add( new Card( s, f ) ) ;

You can use an enumeration valued expression to control a switch statement. For example

    boolean isBlack ;
    switch( s ) {
        case CLUB:
        case SPADE:
            isBlack = true ;
        break ;
        case HEART:
        case DIAMOND:
            isBlack = false ;
    }

Note that, for this switch statement, it is a run-time error for s to be null.

Enumeration types can not be extended.
