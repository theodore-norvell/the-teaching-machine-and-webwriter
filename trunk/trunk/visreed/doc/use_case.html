Project visreed: Use Case

The use cases of the project are listed below.

UC1: Diagram generation from text

Primary Actor:

the user

Stakeholders and Interests:

  • The User: Wants accurate graphical representation of the text input.

Preconditions:

Success Guarantee (Postconditions):

Basic Flow:

  1. The use inputs the text form of the regular expression to the system.
  2. The system parses the regular expression, and apply sytle to it.
  3. The system generates a graphical diagram from the input text, reflecting its lexical structure.
  4. The system generates another graphical diagram from the input text, showing its lexical flow.
  5. The system arrange these two diagrams, together with the text, to a propriate position.

Extensions:

Special Requirements:

Technology and Data Variantions List:

  • Regular expression parsing
  • Generating views for each model

UC2: Diagram re-generation from text

Primary Actor:

the user

Stakeholders and Interests:

  • The User: Wants accurate graphical representation of the modified text input.

Preconditions:

The user has input some text already, and the system has successfully done the UC1.

Success Guarantee (Postconditions):

Basic Flow:

  1. The user modifies the text form of the regular expression to the system.
  2. The user triggers the update action.
  3. The system re-parses the regular expression, and updates the style to it.
  4. The system generates an updated lexical tree diagram / modifies the existing lexical tree diagram from the input text.
  5. The system generates an updated lexical flow diagram / modifies the existing lexical flow diagram from the input text.
  6. The system arrange these two diagrams, together with the text, to a propriate position.

Extensions:

Special Requirements:

Technology and Data Variantions List:

  • Regular expression parsing
  • Generating views for each model
