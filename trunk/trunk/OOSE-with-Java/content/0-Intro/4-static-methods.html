Static Methods

In Java, subroutines come in three varieties:

  • constructors
  • nonstatic methods
  • static methods

We'll look at constructors and nonstatic methods later on.

So this section deals with a variety of subroutine called static methods.

We've already seen an example of a static method. It is the main method. For example

Here the method is set within a class and, in fact, it is a rule in Java that every subroutine must be declared within a class.

The syntax for classes is this

ClassDeclaration --> Modifiers class Identifier { (ClassMember)* }
ClassMember --> MethodDeclaration
Modifiers --> ( Modifier )*
Modifier --> private
Modifier --> public
Modifier --> static

And for methods it is this

MethodDeclaration --> MethodSignature Block
MethodSignature --> Modifiers ReturnType Identifier ( Parameters )
ReturnType --> void
ReturnType --> Type ( [] )*
Parameters --> ( ParameterDecl ( ,ParameterDecl)* )?
ParameterDecl --> Type ( [] )* Identifier

For static methods the modifiers should include the keyword static.

For methods intended to be called from outside the class, we should include the keyword public in the modifiers. All other methods we'll declare (for now) as private.

[There are a few choices besides public and private, we'll expand on this later.]

To call a method declared in the same class we use a method call

Expression --> MethodName ( Arguments )
Arguments --> ( Expression ( ,Expression)* )?

The methods we've seen so far have a return type of void. void isn't really a type at all, it is simply a keyword that indicates that the method returns no result value.
