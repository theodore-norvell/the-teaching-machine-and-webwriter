webWriter Basics

webWriter++ is the tool used to produce these web pages. It works hand in hand with the Teaching Machine, which animates programming examples for you.

webWriter is actually designed for instructors, but since you have to work with the consequences, we though a little help for students would be useful.

Most of your interactions with webWriter will be via the title bar which appears at the top of every page. In addition to the name of your course and the webWriter logo it contains a number of buttons.

Waiting for Applets ...

You will probably see this message the first time webWriter loads. The Teaching Machine program animator is a large applet—that is a web enabled Java program. Although we typically don't need the applet on every page, it is used over most of the site. Since it is large, it can take a long time to load (depending on how fast your connection is). So our strategy is to load it once and then keep it loaded—and that loading is done when the webWriter bar is loaded.

In a real sense, the webWriter title bar is a visible front for the Teaching Machine applet.

Load-Once Consequences

You may find it difficult to book mark individual pages. That is because, before a page is loaded, we must make sure the titleBar (and therefore the Teaching Machine and webWriter) are loaded. While we have put technology in place to manage that automatically, each browser works with it a little differently. Moreover, every time we get it working across all major browsers, an automatic update gets installed and breaks it.
