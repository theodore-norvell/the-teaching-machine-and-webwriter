View Commands

View commands are used to control the appearance of a in one or more . View commands are shipped to all Visualizers on the and none which are not.

View commands are divided up into a number of categories

Default Node Commands

Default commands control the properties of elements (in this case, ) which are not individually specified.

Default Edge Commands

Control the default properties of elements which are not individually specified.

Default Branch Commands

Control the default properties of elements which are not individually specified.

Default Zone and Label Commands

Control the default properties of elements which are not individually specified.

Specified Node Commands

Control the properties of specified . Nodes are specified by using the name of the variable used to create the node in the first place.

Specified Edge Commands

Control the properties of specified . Edges are specified by using the names of the variables used to create the nodes at either end of the edge.

Specified Branch Commands

Control the properties of specified . Branchs are specified by using the names of the child variable associated with the branch.

Deref Versions of View Commands

There are deref versions of any routine which uses datums as arguments. Simple put, an optional boolean can appear after any pointer datum argument to specify that the node is associated with the target of the pointer rather than the pointer itself.

Deref versions of Specified Node Commands

Control the properties of specified. Nodes are specified by using a reference to the object originally used to create the node. Will not work with primitives, such as int, double or boolean since Java passes these by value.

Deref versions of Specified Edge Commands

Control the properties of specified. Edges are always specified by the nodes which define their endpoints (the target node and source node). Commands that specify edges have been overloaded so that either or both end nodes may be defined by a reference to the object originally used to create the end node. Will not work with primitives, such as int, double or boolean since Java passes these by value.

Deref versions of Specified Branch Commands

Control the properties of specified. Branchs are specified by using a reference to the object originally used to create the child node. Will not work with primitives, such as int, double or boolean since Java passes these by value.

String Commands

These allow visualizations to be annotated with arbitrary strings.
